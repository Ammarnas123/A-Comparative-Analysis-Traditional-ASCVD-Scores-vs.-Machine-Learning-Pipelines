Capstone Project Proposal: Evaluating and Enhancing Cardiovascular Disease Prediction using Machine Learning vs. Traditional ASCVD Risk Scores

1. Introduction and Background
Cardiovascular disease (CVD) remains a leading cause of mortality globally. Traditional clinical tools, such as the Atherosclerotic Cardiovascular Disease (ASCVD) risk estimator, use a standard set of variables (age, sex, cholesterol, blood pressure, smoking status) to predict the 10-year risk of a first hard atherosclerotic cardiovascular event. However, Machine Learning (ML) presents an opportunity to capture complex, non-linear relationships between clinical variables that traditional statistical models might miss.
This project aims to leverage the Framingham Heart Study dataset to compare traditional ASCVD risk predictions against actual 10-year patient outcomes. We will objectively evaluate if modern Machine Learning algorithms can outperform traditional predictive equations. To ensure academic rigor, reproducibility, and reliability, the entire lifecycle of this project will follow standard machine learning methodologies.

2. Project Objectives
•	ASCVD Baseline Evaluation: Calculate the 10-year ASCVD risk using traditional equations and measure its deviation from the actual clinical outcomes in the Framingham database.
•	Robust Predictive Modeling: Train, tune, and test various ML algorithms to predict, 10-year CVD onset.
•	Quality-Assured ML Lifecycle: Apply standard ML lifecycle methodologies to ensure data quality, model robustness, and reliable performance evaluation.
•	Comparative Analysis & Feature Importance: Compare ML performance against ASCVD scores and utilize feature selection techniques to identify crucial health indicators.

3. Dataset Overview
•	Data Source: Framingham Heart Study dataset.
•	Sample Size: 4,239 participants.
•	Follow-up Period: 10 years.
•	Target Variable: TenYearCHD (Binary: 1 if the patient developed Coronary Heart Disease within 10 years, 0 otherwise).

4. Methodology: The Machine Learning Lifecycle
To ensure rigorous research standards, our methodology follows the standard process for Machine Learning.

Phase 1: Project & Data Understanding
•	Clinical Objective Identification: Define what constitutes a successful model (e.g., maximizing Recall to avoid missing at-risk patients while maintaining acceptable False-Positive rates).
•	Data Exploration (EDA): Assess the 4,239 records for class imbalances, feature distributions, and correlations.
•	Quality Check: Identify missing data constraints, outliers, and baseline data formatting issues.

Phase 2: Data Preparation & Baseline Calculation
•	Quality-Driven Preprocessing: Handle missing values through rigorous imputation techniques (mean/median/mode or KNN-imputation) and apply standard scaling for distance-based algorithms.
•	Data Splitting: Randomly partition the dataset into a 70% Training / 30% Testing split to ensure models are strictly evaluated on unseen data, preventing data leakage.
•	Traditional ASCVD Calculation: Synthesize the baseline 10-year risk using traditional pooled cohort equations, converting continuous risk scores into binary clinical thresholds (>7.5% or >20%) to measure deviation from actual outcomes.

Phase 3: Model Engineering
•	Feature Engineering & Selection: Utilize Forward Feature Selection and Backward Feature Elimination to iteratively construct an optimal, noise-free subset of predictors.
•	Algorithm Implementation: Train the following models on the 70% training set:
1.	K-Nearest Neighbors (KNN)
2.	Decision Trees (DT)
3.	Random Forest (RF)
4.	Gradient Boosting Machine (GBM) (e.g., XGBoost, LightGBM)
5.	Future Models: Seamlessly integrate newly learned algorithms (e.g., SVM, Neural Networks) as the project progresses.

Phase 4: Model Evaluation 
•	Testing Phase: Evaluate all models on the 30% hold-out test set.
•	Performance Metrics: Standard accuracy is insufficient for imbalanced medical data. Model performance will be evaluated using:
·	Sensitivity (Recall): Maximizing the detection of true CVD cases.
·	Specificity: Correctly identifying healthy patients.
·	F1-Score: Balancing precision and recall.
·	ROC-AUC: Measuring the models' ability to distinguish between classes.
•	Robustness Check: Utilize cross-validation on the training set to ensure the models are not overfitting and that hyperparameter choices are stable.

5. Expected Outcomes and Deliverables
1.	A Cleaned & Preprocessed Dataset: Fully reproducible data pipeline.
2.	Baseline Analytics Report: Quantifying the deviation of traditional ASCVD calculators from actual 10-year realities.
3.	Optimized, Evaluated ML Pipeline: A documented, rigorously tested Python codebase demonstrating the 70/30 split, feature selection, and modeling.
4.	Final Capstone Paper & Presentation: A detailed report highlighting ML vs. traditional clinical tools, critical risk features, and how machine learning methodologies ensured a scientifically valid result.
